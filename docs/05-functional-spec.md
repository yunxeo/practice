## 3. 인증 (Auth)

### 3-1. 회원가입 (FR-001)

| 항목 | 내용 |
|---|---|
| 진입 | 온보딩 화면 → "시작하기" |
| 단계 | 2단계 (기본 정보 → 학교 선택) |

**Step 1 — 기본 정보**

| 입력 필드 | 유효성 규칙 |
|---|---|
| 이메일 | 이메일 형식, 중복 불가 |
| 닉네임 | 2~20자 |
| 비밀번호 | 영문+숫자 조합, 8자 이상 |
| 비밀번호 확인 | 위 비밀번호와 일치 |

**Step 2 — 학교 선택**

- 대학교 검색 후 선택 (선택 사항)
- "건너뛰기" 허용
- 선택 시 해당 학교 교수 피드 우선 노출

**처리 로직**
- 비밀번호: Argon2 해시 후 저장
- 성공 시 Access Token + Refresh Token 발급, 홈으로 이동

---

### 3-2. 로그인 (FR-002)

| 입력 | 유효성 |
|---|---|
| 이메일 | 이메일 형식 |
| 비밀번호 | 8자 이상 |

- 실패 시 "이메일 또는 비밀번호가 일치하지 않아요" 인라인 에러
- 성공 시 Access Token(15m) + Refresh Token(7d) 발급
- Access Token 만료 시 Refresh Token으로 자동 갱신

---

### 3-3. 로그아웃

- 로컬 토큰 삭제 후 온보딩으로 이동

| 교수 목록 | 무한 스크롤 |

## 5. 교수 (Professor)

### 5-1. 교수 검색 (FR-003)

| 검색 파라미터 | 설명 |
|---|---|
| `q` | 교수 이름 부분 일치 |
| `courseName` | **과목명 검색** — 해당 과목을 가르친 교수 반환 |
| `universityId` | 대학교 필터 |
| `departmentId` | 학과 필터 |
| `sortBy` | `rating` (기본) \| `reviews` \| `name` |
| `order` | `desc` (기본) \| `asc` |

**검색 모드 전환**: 홈 화면에서 "교수명 / 과목명" 토글 제공

---

### 5-2. 교수 프로필 조회 (FR-004, US-003)

**표시 정보**

| 섹션 | 내용 |
|---|---|
| 헤더 | 사진, 이름, 직위, 소속 대학·학과 |
| 종합 평점 | 전반 평점 + 별 아이콘 + 후기 수 |
| 평점 상세 | 전반 / 난이도 / 전달력 / 친절도 (막대 그래프) |
| 태그 | 학생들이 가장 많이 선택한 태그 Top 8 |
| 연구 분야 | 교수 등록 데이터 |
| AI 요약 | 접기/펼치기 토글로 제공 |
| 후기 목록 | 좋아요 순 정렬, 무한 스크롤 |

## 6. 후기 (Review)

### 6-1. 후기 작성 (FR-005, US-004)

**입력 항목**

| 필드 | 필수 | 유효성 |
|---|---|---|
| 전반 평점 | ✅ | 1~5 (별점) |
| 강의 난이도 | ✅ | 1~5 |
| 전달력 | ✅ | 1~5 |
| 친절도 | ✅ | 1~5 |
| 후기 내용 | ✅ | 20자 이상, 2000자 이하 |
| 수강 과목명 | ❌ | 최대 100자 |
| 수강 학기 | ❌ | 선택 칩 (최근 5학기) |
| 태그 | ❌ | 최대 5개 (사전 정의된 15개 중 선택) |
| 익명 여부 | ❌ | 기본 false |

**제약**
- 교수 1인당 사용자 1후기 (중복 작성 불가)
- 클라이언트 욕설 필터 (1차), 서버 AI 필터 (2차, Phase 2)

**부작용**
- 제출 후 해당 교수 `avg_rating`, `avg_difficulty`, `avg_clarity`, `avg_helpfulness`, `review_count` 자동 재계산 (DB 트리거)

---

### 6-2. 후기 조회 (FR-006)

| 정렬 옵션 | 기본 |
|---|---|
| 좋아요 순 | ✅ |
| 최신 순 | |

**표시 항목**: 작성자(익명 처리 가능) / 별점 / 과목명·학기 / 본문 / 난이도·전달력·친절도 칩 / 좋아요 / 신고

---

### 6-3. 좋아요

- 로그인 사용자만 가능
- 토글 방식 (좋아요 ↔ 취소)
- 중복 불가 (DB Unique 제약)

---

### 6-4. 후기 수정 / 삭제

- **수정**: 본인만 가능, 모든 필드 부분 수정 가능
- **삭제**: 본인 또는 admin, Soft Delete (`is_deleted = true`)

## 7. AI 분석 (AI Analysis)

### 7-1. 교수 AI 요약 (FR-007, US-005)

| 항목 | 내용 |
|---|---|
| 트리거 | 교수 상세 → "AI 요약 보기" 토글 |
| 인증 | 로그인 필요 |
| 캐싱 | `ai_profiles` 테이블에 24시간 캐시 |
| 재생성 조건 | 캐시 만료 OR 후기 수 5개 이상 증가 |
| 프로바이더 | OpenAI GPT (키 없을 시 Mock 자동 대체) |

**요약 내용 포함 요소**
- 강의 스타일 요약
- 평가 방식 성향
- 학생 평균 반응
- 수강 시 참고사항

**미래 확장**
- `ai_profiles.persona` 필드: 교수 페르소나 텍스트 저장
- `ai_profiles.analysis_data` 필드: JSON 형태 원시 분석 데이터

## 8. 신고 (Report)

### 8-1. 후기 신고

| 항목 | 내용 |
|---|---|
| 진입 | 후기 카드 → "신고" 버튼 (본인 후기 제외) |
| 인증 | 로그인 필요 |
| 중복 | 동일 후기 중복 신고 불가 |

**신고 사유**

| 코드 | 표시 문구 |
|---|---|
| `spam` | 스팸/광고 |
| `inappropriate` | 욕설/부적절한 내용 |
| `false_info` | 허위 정보 |
| `other` | 기타 |

**상세 내용**: 선택 입력 (500자 이하)

---

### 8-2. 신고 처리 (Admin)

| 상태 | 설명 |
|---|---|
| `pending` | 접수됨 (기본) |
| `reviewed` | 처리 완료 |
| `dismissed` | 반려 |

Admin은 `/reports` GET으로 목록 조회, PATCH `/reports/:id/status`로 상태 변경

## 11. 비기능 요구사항

### 성능

| 지표 | 목표 |
|---|---|
| API 응답 시간 | 500ms 이하 |
| 검색 응답 시간 | 1초 이하 |
| 이미지 로딩 | Lazy load |

### 보안

| 항목 | 구현 |
|---|---|
| 비밀번호 암호화 | Argon2 |
| 인증 | JWT (Access 15m + Refresh 7d) |
| API Rate Limit | 분당 100회 (ThrottlerGuard) |
| CORS | 허용 오리진 제한 |
| 욕설 필터 | 클라이언트 1차, 서버 2차 (Phase 2) |

### 확장성

- 사용자 10만 명까지 Supabase 기본 구성으로 처리 가능
- AI 프로바이더 교체 용이 (`AiProviderInterface` 추상화)
- Phase 2: Meilisearch 전환으로 검색 고도화

## 13. 데이터 모델 요약

```
users           — 회원 계정
universities    — 대학교 마스터
departments     — 학과 (university 하위)
professors      — 교수 프로필 + 평점 집계
reviews         — 학생 후기 (professor + user)
review_likes    — 후기 좋아요
professor_tags  — 교수 태그 집계
ai_profiles     — AI 요약 결과 (professor 1:1)
reports         — 신고 (review 기반)
```
